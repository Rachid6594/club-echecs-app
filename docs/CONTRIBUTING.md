# Contribution

## Travail par sprint

Le developpement se fait uniquement par sprint. Un sprint doit etre valide avant de commencer le suivant.

## Branches

Format obligatoire :

```text
sprint/XX-nom-du-sprint
```

## Commits

Format recommande :

```text
feat(sprint-XX): description claire
```

## Definition of Done

- Perimetre du sprint implemente.
- Tests executes et documentes.
- Bugs critiques corriges.
- Documentation mise a jour.
- Validation Markdown presente.
- Aucun secret commite.
- Sprint pret pour pull request.
